Best Practices When Using Classes and ID's

What is a class and what is an id?

When building a website with html and CSS, you have a myriad of ways to create the content and style the content. There's a symbioticrelationship between Html and CSS

Classes and ID’s are CSS attributes you can use to add styling to your webpage. While they perform very similar functions, there are specific scenarios when you should be using a class and vice versa for an ID.

What is a class?

The class attribute can be used to style many elements. You can create a class which essentially gives styling instructions to each element that uses the class.

To create a class in CSS, you use a "." before the name of the class. As an example, .header is creating a class named "header".

What is an ID?

While Classes can be used to style a Class of elements, id’s are unique and can only be used to style one element. While classes can be used for to style many (unlimited) elements on a in html, only one id can be used per id in the html file

To create a id in CSS, you use a "#" before the name of the id. As an example, #header is creating an id named "header". This id can only be used for one element.

Analogy

As a former bank teller, an analogy I’ll use for Classes and ID’s are bank account numbers.

In New Zealand we have 4 sections for our bank account numbers:

Bank Prefix - Branch - Account number - Account Suffix.

As an example: 02-0100-0123456-00

Classes:

02 represents the bank - BNZ

0100 represents the branch - Queen Street, Auckland

ID:

0123456 is the customer number which is unique to each customer. No two customers will have the same customer number.

For example, anybody that set up their bank account at the BNZ Queen Street branch will have the bank-branch 02-0100 but only one customer will have the 02-0100-0123456.

Side-note: The last class is the Account Suffix -00 which, dependent on the bank, often represents the account type (i.e. 00 is often an everyday account, 66 is often for a savings account).
