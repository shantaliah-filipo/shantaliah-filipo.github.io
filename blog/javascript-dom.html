Sprint 3 - Introduction to Javascript

An analogy to describe Javascript and it’s relationship to HTML and CSS

The makeup of websites are often created by three aspects: HTML, CSS, and Javascript. An analogy for the relationship between HTML, CSS, and Javascript is the human body where:

  • The HTML is the skeleton
  • The CSS is the skin
  • The Javascript are the muscles.

An HTML file of a website contains the written content and structure of the page itself. Like a skeleton, it is the foundation of the body of a website.

A CSS file contains the styling or decoration of the page. It’s where you can alter aspects such as colour schemes, fonts, positioning and layouts. It’s essentially the skin covering the skeleton.

A JavaScript file determines the dynamic and interactive elements on the page. This includes actions such as what happens when users click, hover, or type within certain elements of a website, however the opportunities for interactivity are endless. You can think of Javascript as the muscles on a human body as they are what allow for movement of the body.

Control flow and loops

In Javascript, the control flow is how the computer runs or executes code from top to bottom. Like watching a TV show for the very first-time, you start from the beginning and watch it in sequential order (unless you’re a weirdo). In Javascript, the computer starts reading from the first line and finishes at the last line. However, while this is the case, there are statements in Javascript such as functions, conditionals, and loops that can alter the control flow (i.e. how the computer runs the code). Loops are conditional statements that repeat a given task until a given condition is met. In other words, it’s a sequence of instruction that is continually repeated until a certain condition is reached. In other words, a loop keeps looping until it can’t loop no more (based on the given condition). There’s three parts to a loop: the loop variable, the condition, and the increment expression. We won’t go into too much details about what these mean at this stage. Let’s use the example of making a bowl of cereal to explain the relationship between control flow. Based on the control flow, you could get your variables which would be your bowl, cereal, milk. You could set a variety of loops for this activity (i.e. pour cereal into the bowl UNTIL it’s halfway, pour milk into the bowl UNTIL it covers the cereal). Once these conditions have been met, the control flow can continue (and then you can eat your bowl of cereal.

What is the DOM?

As we’ve already learned, a website is made up of these 3 elements: HTML, CSS, and Javascript. These 3 languages works together to make things happen. Without CSS, the HTML just looks like a word document, and without Javascript, the HTML and CSS will just look like a prettied up brochure with no interactivity. The Document Object Model (DOM) is the entire data representation of the objects that comprise the structure and content of a document on the web. According to freeCodeCamp, “It is a programming interface that allows us to create, change, or remove elements from the document. We can also add events to these elements to make our page more dynamic. The DOM views an HTML document as a tree of nodes. A node represents an HTML element.” In short, the DOM is the glue that brings HTML, CSS, and Javascript together. The (DOM) is a way that we can change the HTML with Javascript.

What are functions?

A JavaScript function is a block of code designed to perform a particular task. A JavaScript function is executed when "something" invokes it (calls it). To return a value other than the default, a function must have a return statement that specifies the value to return. Functions are useful as they allow you to define a block of code and then run it as many times as you desire.
